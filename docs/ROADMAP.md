# ROADMAP

각 버전은 이전 버전이 동작하는 상태에서 시작한다. 한 버전 작업 중에는 다음 버전 범위의 코드를
미리 만들지 않는다. 버전 완료 기준(Definition of Done)을 만족하면 커밋하고 다음 단계로 넘어간다.

## v0 — 프로젝트 초기화

- Next.js + TypeScript + Tailwind 프로젝트 스캐폴딩 (완료: `create-next-app` 기반).
- `CLAUDE.md`, `docs/ROADMAP.md`, `docs/GESTURES.md` 작성.
- lint/build 통과하는 빈 페이지 상태 확인.

**완료 기준:** `npm run dev`, `npm run build`, `npm run lint` 가 모두 에러 없이 통과한다.

## v1 — 웹캠 화면 출력

- `getUserMedia` 로 웹캠 스트림을 요청하고 `<video>` 엘리먼트에 실시간으로 출력한다.
- 카메라 권한 거부/미지원 브라우저에 대한 최소한의 안내 UI.
- 웹캠 컴포넌트는 `components/webcam/` 에 위치.

**완료 기준:** 브라우저에서 카메라 권한을 허용하면 실시간 웹캠 화면이 페이지에 보인다.

## v2 — MediaPipe Hand Landmarker 연결

- `@mediapipe/tasks-vision` 패키지 도입 (사전 승인 필요).
- `lib/mediapipe/` 에 HandLandmarker 초기화/모델 로드 로직 작성.
- 비디오 프레임마다 추론을 실행해 랜드마크 결과를 얻는다 (아직 화면에 그리지 않아도 됨, 콘솔 로그 수준).

**완료 기준:** 웹캠 앞에서 손을 움직이면 콘솔(또는 임시 디버그 패널)에 21개 랜드마크 좌표가
실시간으로 갱신된다.

## v3 — 랜드마크 오버레이

- 비디오 위에 겹치는 `<canvas>` 를 두고, 감지된 손 랜드마크(점 + 연결선)를 실시간으로 그린다.
- `components/landmarks/` 에 렌더링 로직 작성. 비디오 해상도와 캔버스 좌표계를 일치시킨다.

**완료 기준:** 웹캠 화면 위에 손 스켈레톤(점과 선)이 실시간으로 오버레이되어 움직임을 따라간다.

## v4 — 제스처 감지 엔진

- `lib/gestures/` 에 [docs/GESTURES.md](GESTURES.md) 에 정의된 제스처별 판정 순수 함수 구현.
- 프레임 단위 랜드마크를 판정 함수에 흘려보내는 파이프라인(감지 → 상태 유지/쿨다운 → 이벤트 발행) 구성.
- 아직 실제 이펙트는 없어도 되며, 제스처 발동 시 화면에 텍스트/로그로 표시.

**완료 기준:** 여우 소환 / 핀 뽑기 변신 / 체인 리코일 변신 / 건 제스처(Bang) 네 제스처가
각각 안정적으로(오탐 최소화) 감지되고 화면(또는 콘솔)에 어떤 제스처가 발동했는지 표시된다.

## v5 — 이펙트 발동

- 제스처 이벤트에 반응하는 애니메이션 이펙트 구현 (`components/effects/`).
- 제스처별로 시각적으로 구분되는 이펙트를 연결 ([docs/GESTURES.md](GESTURES.md) 의 "시각
  이펙트 개요" 참고: 여우 소환 → 여우 실루엣 머리 + 소환 심볼 텍스트, 핀 뽑기 변신 → 목 폭발
  파티클 + 헤드웨어, 체인 리코일 변신 → 가슴 스파크 + 체인/톱날 트래킹, 건 제스처 → 충격파
  레이저 + 화면 흔들림).
- 여우 머리 일러스트/3D 모델, 헤드웨어, 체인/톱날 이펙트 등 오리지널 아트 에셋 소싱/제작
  (직접 제작 또는 라이선스가 명확한 리소스만 사용, [docs/GESTURES.md](GESTURES.md) 의
  "저작권 관련 메모" 참고).
- 쿨다운/중복 발동 방지 등 v4의 감지 엔진과 연동.

**완료 기준:** 네 제스처 각각에 대해 시각적으로 명확히 구분되는 이펙트가 실시간으로 재생된다.

## v6 — 포트폴리오용 README 정리

- 프로젝트 소개, 데모 스크린샷/GIF, 기술 스택, 실행 방법, 제스처 사용법을 담은 `README.md` 정리.
- 알려진 한계(조명, 카메라 각도, 브라우저 호환성 등) 명시.
- 배포(예: Vercel) 여부 결정 및 반영.

**완료 기준:** 처음 보는 사람이 README만 보고 프로젝트를 실행하고 제스처를 시연해볼 수 있다.
